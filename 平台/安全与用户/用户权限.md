# 用户与权限

本文档归档 **j2agent** 的用户数据模型、登录会话、角色鉴权与用户管理 REST API。邮箱自助注册见 [邮箱注册机制.md](./邮箱注册机制.md)。

## 用户模型

### 数据表

库表 `j2agent.user`（[`V0_0__schemas.sql`](../../j2agent/j2agent-server/src/main/resources/sql/migration/mysql/zh_CN/V0_0__schemas.sql)）：

| 列 | 类型 | 说明 |
|----|------|------|
| `id` | `char(32)` PK | UUIDv7（[`UUIDv7Utils`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/utils/UUIDv7Utils.java)） |
| `username` | `varchar(64)` NOT NULL | 登录名 |
| `password_hash` | `char(64)` NOT NULL | `SHA256(userId + password)` |
| `create_time` | `bigint` | 创建时间戳 |
| `role` | `int` | `1` 管理员，`2` 普通用户 |
| `email` | `varchar(128)` NULL | 邮箱；唯一索引 `uq_user_email` |

密码算法见 [`UserUtil`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/utils/UserUtil.java)：`getPasswordHash(userId, password)` / `verifyPassword`。

### 内置管理员

Flyway [`V0_1__init_data.sql`](../../j2agent/j2agent-server/src/main/resources/sql/migration/mysql/zh_CN/V0_1__init_data.sql) 插入：

- `username` = **`aiadmin`**
- `role` = `1`（管理员）
- 固定 `id`：`AZ5EsB1JcMqXv6z1FcfEUw`

[`UserService`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/service/security/UserService.java) 中常量 `BUILTIN_ADMIN_USERNAME = "aiadmin"`，对该用户：

- 不可删除
- 不可修改角色
- 其他管理员不可为其重置密码（`updateUserPassword` 中 `ensureMutableUser`）

## 角色与鉴权

### 角色枚举

[`UserContextBo.RoleEnum`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/model/security/UserContextBo.java)：

| 枚举 | 值 | 说明 |
|------|-----|------|
| `ADMIN` | 1 | 管理员 |
| `USER` | 2 | 普通用户 |

**数值越小权限越高。** 判断能否访问某接口要求：

```text
userContext.role.value <= requiredRole
```

例如：要求 `ADMIN(1)` 时，仅 `role=1` 通过；要求 `USER(2)` 时，管理员与普通用户均可通过。

`userContext.isAdmin()` 等价于 `hasAccess(ADMIN)`。

角色与 `permissions`（预留细粒度权限）存储在 Redis，由 [`UserLoginContextCache`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/service/security/UserLoginContextCache.java) 管理，**不写入 JWT**。

### @RequiredRole 注解

[`RequiredRole`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/config/security/RequiredRole.java) 标注在 Controller 类或方法上：

| 常量 | 值 | 含义 |
|------|-----|------|
| `RequiredRole.ADMIN` | 1 | 仅管理员 |
| `RequiredRole.USER` | 2 | 已登录即可（含管理员） |

未标注时，[`LoginInterceptor`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/interceptor/LoginInterceptor.java) 默认按 `USER(2)` 校验。

### LoginInterceptor

配置于 [`InterceptorConfig`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/config/web/InterceptorConfig.java)：

- 拦截：`/v*/**`
- 排除：`/v*/auth/**`（登录、验证码、邮箱注册等公开接口）

流程：

1. [`LoginInterceptor`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/interceptor/LoginInterceptor.java) 调用 `LoginService.resolveRequest`：从 `Authorization: Bearer` 或查询参数 `authorization` 读取 JWT，验签并 `bindSession`。
2. 未登录：**401**，并 `clearSession`。
3. 已登录但角色不足：**403**，并 `clearSession`。
4. 请求结束：`afterCompletion` 中 `clearSession`，避免线程池复用导致会话串号。

WebSocket 握手通过查询参数 `authorization` 传递 token，由 [`WebsocketLoginInterceptor`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/interceptor/WebsocketLoginInterceptor.java) 校验。

## 登录与 JWT

### 登录

- **接口**：`POST /v1/auth/j2agent/login`（[`LoginController`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/controller/LoginController.java)）
- **请求体**：`username`、`password`、`validateCode`、`hash`（滑动验证码，与注册发码相同机制）
- **成功**：`200` + `AuthResultDto`（`token`、`expiresIn`），JWT 与外部系统（`platsvc-security` / `platsvc-gateway`）共用密钥与 claims 结构（配置前缀 `j2agent.security.jwt`）
- **失败**：401

### JWT（登录凭证，与外部系统算法互通）

登录签发、请求验签均通过 [`JwtService`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/jwt/JwtService.java) 完成，配置前缀 `j2agent.security.jwt`，算法与 claims 与外部系统（`platsvc-security` / gateway）一致。**JWT 仅负责登录校验（验签 + 身份标识），role/permissions 从 Redis 读取。**

| Claim | 说明 |
|-------|------|
| `user` / `name` | 用户 ID / 用户名 |
| `sid` | 会话标识，用于 Redis 缓存键 |
| `terminal` | `BROWSER` |
| `classify` | `system` |
| `auth` | `auth-pc`（外部系统终端鉴权） |

| 行为 | 说明 |
|------|------|
| 登录 | 返回 `{ token, expiresIn }`；同时将 `UserContextBo`（含 role、permissions）写入 Redis |
| 鉴权 | 验签 JWT → 按 `sid` 读 Redis；缓存 miss 或过期则从 DB 重建，TTL = JWT 剩余有效期 |
| 登出 | 客户端丢弃 token；服务端删除 `login:ctx:{sid}` |
| 改密/删用户/改角色 | `invalidateUserLogin` 删除该用户全部 sid 缓存；下次请求从 DB 重建 |

### Redis 用户上下文缓存

[`UserLoginContextCache`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/service/security/UserLoginContextCache.java) 使用 `StringRedisTemplate` + JSON 序列化 `UserContextBo`：

| Key | 类型 | 值 | TTL |
|-----|------|-----|-----|
| `{appName}:login:ctx:{sid}` | String | `UserContextBo` JSON（含 role、permissions） | JWT `expiresIn`（登录时）或 JWT 剩余秒数（重建时） |
| `{appName}:login:user-sids:{userId}` | Set | 该用户活跃 `sid` 集合 | 与 ctx 同步刷新 |

**外部系统互通**：`j2agent.security.jwt` 的 `algorithm`、`secret`、`kmac-output-length-bytes` 须与外部系统（`com.nms.jwt` / `platsvc-security`）**完全一致**，否则验签失败返回 401。开发环境可通过环境变量覆盖：

- `J2AGENT_JWT_SECRET` — 与外部系统相同的密钥（支持 `base64:` 前缀）
- `J2AGENT_JWT_KMAC_OUTPUT_LENGTH_BYTES` — 与外部系统签发时一致（常见 `64` 或 `256`）

### 外部系统 SSO 跳转

前端与外部系统共用 localStorage 键 `access_token`。跳转方式：

| 场景 | URL 示例 |
|------|----------|
| 同域（共享 localStorage） | `https://host/j2agent/#/agents` |
| 跨域 / 不同端口 | `https://host/j2agent/?authorization=<jwt>#/agents` |

启动时 [`consumeLaunchQueryFromUrl`](../../j2agent-ui/src/utils/launchQuery.ts) 统一解析并剥离 URL 启动参数（当前含 `authorization` → `access_token`；新参数在此模块扩展）；[`bootstrapSessionFromToken`](../../j2agent-ui/src/utils/auth.ts) 带 Bearer 调 `GET /v1/rest/j2agent/session`，触发后端验签与自动注册。已在 `/login` 且 token 有效时自动跳主页。

**注意**：j2agent 鉴权失败时**不会**删除 `access_token`，避免破坏外部系统会话；仅 [`Logout.vue`](../../j2agent-ui/src/pages/login/Logout.vue) 主动登出时清除。

**排障**：若跳转后仍 401，优先检查 JWT `secret` 与 `kmac-output-length-bytes` 是否与外部系统一致；用浏览器 Network 查看 `/session` 响应码。

### 外部 Token 自动注册

外部系统携带**合法 JWT** 访问时，若本库不存在 JWT `user` claim 对应的主键，[`UserService.provisionExternalUser`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/service/security/UserService.java) 在 [`LoginService.loadAndCacheContext`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/service/security/LoginService.java) 中自动建档：

| 字段 | 规则 |
|------|------|
| `id` | 直接沿用 JWT claim `user`（trim 后非空且长度 ≤ 32） |
| `username` | 来自 JWT claim `name`；为空则回退为 `userId`；若用户名已占用则追加 `_{6位随机小写字母+数字}` |
| `role` | 固定 `2`（普通用户） |
| `password_hash` | 随机密码哈希（仅 SSO/JWT 登录，不可本地密码登录） |

**不再**按 `name` 回退匹配本地已有用户名，避免外部 token 误绑本地同名账号。`user` claim 缺失或非法时仍返回 401。

Redis miss 时流程：验签通过 → 按 `user` 查 DB → 不存在则自动注册 → 写入 Redis 缓存。

### 当前会话查询

- **接口**：`GET /v1/rest/j2agent/session`（[`SessionController`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/controller/SessionController.java)）
- **请求头**：`Authorization: Bearer <token>`
- **响应**：`SessionInfoDto`（`userId`、`username`、`role`）
- **未登录**：401

### 登出

- **接口**：`GET /v1/auth/j2agent/logout`
- **请求头**：建议携带 `Authorization: Bearer <token>`，服务端按 `sid` 删除 Redis 缓存；客户端同时清除 token

### 路径与权限矩阵

| 区域 | 路径前缀 | 需要登录 | 典型角色要求 |
|------|----------|----------|----------------|
| 认证公开 | `/v1/auth/**` | 否 | — |
| 业务 REST | `/v1/rest/**` | 是 | 见各接口 `@RequiredRole` |
| 用户管理 | `/v1/rest/j2agent/users` 等 | 是 | 见下节 |

**类级别 `@RequiredRole(ADMIN)` 的控制器**（整类接口仅管理员）：

| 控制器 | 说明 |
|--------|------|
| `KnowledgeController` | 知识库 |
| `PropertyController` | 系统属性（含邮箱注册开关等） |
| `McpController` | MCP |
| `EmbeddingsController` | 向量 |
| `AgentPluginController` | Agent 插件 |
| `HealthCheckController` | 健康检查 |

**未标注或方法级标注**：如 `ChatController` 等，默认 **已登录普通用户** 即可；`UserController` 在方法上区分 ADMIN / USER。

服务层 [`UserService`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/service/security/UserService.java) 对管理操作另有 `requireAdmin()`，与拦截器形成 **双重校验**（拦截器为主，Service 为防御性检查）。

## 用户管理 API

实现：[`UserController`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/controller/UserController.java) + [`UserService`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/service/security/UserService.java)。

OpenAPI：[`openapi-interface.yaml`](../../j2agent/j2agent-model/src/main/resources/openapi-interface.yaml)（`User` tag）。

| 方法 | 路径 | @RequiredRole | 说明 |
|------|------|---------------|------|
| GET | `/v1/rest/j2agent/users` | ADMIN | 用户列表，`create_time DESC, username ASC` |
| POST | `/v1/rest/j2agent/users` | ADMIN | 创建用户；body：`username`、`password`、可选 `role`（默认 USER） |
| DELETE | `/v1/rest/j2agent/users?user-id=` | ADMIN | 删除用户；内置 `aiadmin` 不可删 |
| PUT | `/v1/rest/j2agent/users/role` | ADMIN | 修改角色；`userId` + `role`（仅 1 或 2） |
| PUT | `/v1/rest/j2agent/users/password` | USER | 修改密码；见下表 |

### 修改密码规则

请求体 `UserPasswordUpdateRequestDto`：`userId`（可选）、`newPassword`。

| 操作者 | 目标 | 规则 |
|--------|------|------|
| 普通用户 | 自己（`userId` 省略或等于当前用户） | 允许修改 |
| 普通用户 | 他人 | **403** |
| 管理员 | 自己 | 允许修改 |
| 管理员 | 他人 | 允许重置；**不可**修改 `aiadmin` |

改密、删用户、改角色后，[`UserService`](../../j2agent/j2agent-server/src/main/java/io/github/jerryt92/j2agent/service/security/UserService.java) 调用 `invalidateUserLogin` 清除 Redis 缓存；旧 JWT 在 `exp` 到期前仍可通过验签，但下次请求会从 DB 读取最新 role（用户已删则 401）。

## 用户创建途径对比

| 途径 | 接口 / 入口 | 角色 | 说明 |
|------|-------------|------|------|
| 迁移脚本 | SQL | ADMIN | 仅 `aiadmin` |
| 管理员创建 | `POST .../users` | 可指定 1 或 2 | 需管理员登录 |
| 邮箱自助注册 | `POST .../register` | 固定 USER(2) | 见 [邮箱注册机制.md](./邮箱注册机制.md) |
| 外部 JWT 首次访问 | 携带合法 Bearer token | 固定 USER(2) | JWT `user` 不在 DB 时自动注册，见上文 |

自助注册用户可用邮箱（或注册时指定的 `username`）登录；`email` 字段写入 `user.email`。
