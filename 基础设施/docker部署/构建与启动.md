# 构建与启动

## 1. 后端打包

在仓库根目录执行（与 [`docker/j2agent/Dockerfile`](../../j2agent/docker/j2agent/Dockerfile) 注释一致）：

```bash
mvn -f j2agent/pom.xml -pl j2agent-model -am -DskipTests install
mvn -f j2agent/pom.xml -pl j2agent-starter -am -DskipTests package
```

产物：`j2agent-starter/target/j2agent-*.tar.gz`，解压后为 `j2agent/classes`、`j2agent/lib` 等，**不含** `dist/`。

## 2. 构建并启动 Compose

```bash
cd j2agent/docker
cp .env.example .env   # 首次
# 编辑 .env
docker compose up -d --build
```

- `j2agent` 镜像名：`j2agent:${TAG}`
- 构建上下文：仓库根目录；Dockerfile：`docker/j2agent/Dockerfile`

## 3. 关键环境变量

| 变量 | 说明 |
|------|------|
| `SPRING_OPTIONS` | 必须包含 `--spring.profiles.active=product`，否则静态资源走 `classpath:/ui/`，宿主机 `ui` 卷不生效 |
| `J2AGENT_PORT` | 应用端口，需与端口映射一致 |
| `J2AGENT_VOLUMES_PATH` | 宿主机数据根目录 |
| `I18N` | 种子数据与 Flyway 语言目录（如 `zh_CN` → `sql/data/mysql/zh_CN.sql`、`sql/migration/mysql/zh_CN`） |
| `JAVA_OPTS` | JVM 参数 |
| `MYSQL_ROOT_PASSWORD` / `MYSQL_DATABASE` | MySQL 仅创建空库名（默认 `j2agent`），表结构由应用初始化 |
| `ETCD_IMAGE_TAG` / `ETCD_PORT` | Milvus 外置 etcd 镜像与宿主机映射端口 |

智能体全局配置不在 Docker 环境变量中维护，请在管理界面 **设置 → 智能体全局配置** 中编辑并保存（持久化至 `ai_properties.agent-global-config-json`）。

完整列表见 [`docker/.env.example`](../../j2agent/docker/.env.example)。

Milvus 依赖外置 `etcd` 服务健康后再启动（`depends_on: service_healthy`）。

## 3.1 数据库初始化（首次空库）

MySQL 容器首次启动时，`MYSQL_DATABASE` 只会创建**空数据库**，不会执行任何 SQL 文件。

`j2agent` 在 Flyway 迁移之前会自动检测空库（无 `api_key_info` 表），并按顺序执行：

1. `sql/schema/mysql/schemas.sql` — 建表
2. `sql/data/mysql/${I18N}.sql` — 种子数据（与 `.env` 中 `I18N` 一致）
3. Flyway — `sql/migration/mysql/${I18N}/` 下增量脚本（如 `V0_1__agent_global_config.sql` 插入 `agent-global-config-json`）

因此 **无需** 在 Compose 中挂载 `docker-entrypoint-initdb.d` SQL。若需完全重建库，停止栈后删除宿主机 `${J2AGENT_VOLUMES_PATH}/volumes/mysql/data`，再重新 `docker compose up`。

## 4. 启动后检查

```bash
docker compose ps
docker compose logs -f j2agent
docker compose exec j2agent sh -c 'echo $SPRING_OPTIONS'
curl -s -o /dev/null -w "%{http_code}\n" "http://127.0.0.1:${J2AGENT_PORT}/index.html"
```

`SPRING_OPTIONS` 应含 `product`；`index.html` 返回 200 表示 `volumes/j2agent/ui` 已挂载且可读。

首次空库启动时，日志中应出现 `Applying SQL bootstrap` 与 Flyway `Successfully applied`。

## 5. 仅更新后端

修改 Java 代码后重新 `mvn package`，再：

```bash
cd j2agent/docker
docker compose up -d --build j2agent
```

前端仅更新 `dist` 目录时**不必**重建镜像，见 [前端静态资源更新.md](./前端静态资源更新.md)。

## 6. 停止与清理

```bash
docker compose down          # 停止并删除容器，保留卷数据
docker compose down -v       # 若 compose 定义了 named volume 会一并删除（当前主要为 bind mount，数据在宿主机 J2AGENT_VOLUMES_PATH）
```
