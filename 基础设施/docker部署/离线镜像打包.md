# 离线镜像打包

脚本位置：[`docker/package_offline.sh`](../../j2agent/docker/package_offline.sh)

用于在无外网的目标机上部署：在**有网络的机器**上拉取/构建 Compose 所需镜像并导出为 `.tar`，再到目标机 `docker load` 后 `docker compose up`。

## 使用步骤

1. 进入 `docker` 目录，确保已存在 `docker-compose.yml` 及正确的 `.env`（或先 `cp .env.example .env`）。

2. 赋予执行权限并运行：

   ```bash
   cd j2agent/docker
   chmod +x package_offline.sh
   ./package_offline.sh
   # 或指定 compose 文件：./package_offline.sh docker-compose.yml
   ```

3. 按提示选择：

   - 目标架构：`linux/amd64` 或 `linux/arm64`
   - 要打包的镜像序号（回车表示全部）
   - 是否跳过全局拉取（默认优先使用本地已有镜像）
   - 导出方式：单个合并 `images_offline_<arch>.tar`，或分文件目录 `images_offline_<arch>_dir/`

4. 将生成的 tar 或目录拷贝到离线服务器，按脚本结束时的提示导入，例如：

   ```bash
   docker load -i images_offline_amd64.tar
   cd j2agent/docker
   docker compose up -d
   ```

## 离线部署 j2agent 的推荐流程

`j2agent` 在 compose 中为 **本地 build** 服务。离线场景通常**不**导出已构建的应用镜像，而是：

1. 用 `package_offline.sh` 拉取并导出中间件镜像，以及 Dockerfile 基镜像（列表中标注 **`(build)`**，如 `eclipse-temurin:21-jre (build)`）。
2. 将 **`j2agent-*.tar.gz` 放到 `docker/j2agent/`**（与 Dockerfile 同级）。可在有 Maven 的机器打包后拷贝，无需在离线机执行 `mvn`。
3. 将 `docker/` 目录（含 `docker-compose.yml`、`.env`、`j2agent/Dockerfile`、同级 tar.gz）与离线镜像 tar 一并传到目标机。
4. 目标机 `docker load` 后，在 `docker/` 目录执行 `docker compose up -d --build`，由 Dockerfile 解压同级 tar.gz 构建 `j2agent:${TAG}` 镜像。

详见 [构建与启动.md §1.1](./构建与启动.md#11-镜像构建时如何找到-targz)。

## 注意事项

- 脚本对 **build** 服务会解析 Dockerfile 的 `FROM` 基镜像并标注 `(build)`，不会把 `j2agent:${TAG}` 当作需拉取的远程镜像。
- 离线机构建 `j2agent` 前，须确保同级目录已放置 `*.tar.gz`，且基镜像（如 `eclipse-temurin:21-jre`）已通过 `docker load` 导入。
- 离线环境仍需准备 **宿主机数据目录**（`ui`、`knowledge-repo` 等），镜像不包含前端与知识库内容。
- PostgreSQL 首次启动会按 `.env` 初始化库名与用户密码；数据目录若已存在则不会重复初始化。
