# 目录与数据卷

Compose 通过环境变量 `J2AGENT_VOLUMES_PATH`（默认 `/opt/j2agent`）将宿主机目录挂载进容器。路径分隔符由 `DIR_PATH_SEPARATOR` 控制（Linux 为 `/`）。

宿主机数据统一放在 **`${J2AGENT_VOLUMES_PATH}/volumes/`** 下，与 [`docker/docker-compose.yml`](../../j2agent/docker/docker-compose.yml) 一致。

## j2agent 服务挂载

| 宿主机路径 | 容器路径 | 用途 |
|------------|----------|------|
| `${J2AGENT_VOLUMES_PATH}/volumes/j2agent/ui` | `/opt/j2agent/volume/ui` | 前端 SPA 构建产物（index.html、assets/） |
| `${J2AGENT_VOLUMES_PATH}/volumes/j2agent/logs` | `/opt/j2agent/volume/logs` | 应用日志、OOM heap dump |
| `${J2AGENT_VOLUMES_PATH}/volumes/j2agent/knowledge-repo` | `/opt/j2agent/volume/knowledge-repo` | Markdown/AsciiDoc 知识库仓库 |
| `${J2AGENT_VOLUMES_PATH}/volumes/j2agent/plugins` | `/opt/j2agent/volume/plugins` | 插件根目录（与 `j2agent.plugin.path` 一致） |
| `${J2AGENT_VOLUMES_PATH}/volumes/j2agent/plugins/agents` | `/opt/j2agent/volume/plugins/agents` | 智能体插件：每个 Agent 解压为一级子目录（瘦 JAR + `resources/`） |
| `${J2AGENT_VOLUMES_PATH}/volumes/j2agent/plugins/skills` | `/opt/j2agent/volume/plugins/skills` | 平台共享 Skill（可选单独挂载） |

配置项对应关系（`application-product.yaml`）：

- `j2agent.knowledge.repo.root-path` → `knowledge-repo`
- `j2agent.plugin.path` → `plugins`（Agent 在 `plugins/agents/`，Skill 在 `plugins/skills/`）
- `spring.web.resources.static-locations` → `file:/opt/j2agent/volume/ui/`

## 依赖服务数据目录

| 服务 | 宿主机路径（示例） | 说明 |
|------|-------------------|------|
| MySQL | `${J2AGENT_VOLUMES_PATH}/volumes/mysql/data` | 数据文件 |
| MySQL | `.../volumes/mysql/conf/my.cnf` | 自定义配置（需事先准备） |
| MySQL | `.../volumes/mysql/run` | socket 目录 |
| Redis | `${J2AGENT_VOLUMES_PATH}/volumes/redis/data` | AOF 持久化 |
| etcd | `${J2AGENT_VOLUMES_PATH}/volumes/etcd` | Milvus 元数据（外置 etcd） |
| Milvus | `${J2AGENT_VOLUMES_PATH}/volumes/milvus/volumes/milvus` | 向量与本地存储数据 |
| Milvus | `${J2AGENT_VOLUMES_PATH}/volumes/milvus` | 可选覆盖配置 `user.yaml` |

## 推荐初始化目录

```bash
BASE="${J2AGENT_VOLUMES_PATH:-/opt/j2agent}"
mkdir -p "$BASE/volumes/j2agent"/{ui,logs,knowledge-repo,plugins/agents,plugins/skills}
mkdir -p "$BASE/volumes/mysql"/{data,conf,run}
mkdir -p "$BASE/volumes/redis/data"
mkdir -p "$BASE/volumes"/{etcd,milvus/volumes/milvus}
```

## 注意事项

1. **不要删除并重建已挂载的顶层目录**（如在挂载点外 `rm -rf ui && mkdir ui`），否则 bind mount 可能仍指向旧 inode，容器内看不到新文件。应在**已有目录内**同步文件，见 [前端静态资源更新.md](./前端静态资源更新.md)。
2. 后端 JAR/镜像**不包含**前端构建产物，首次部署须将 SPA 放入 `volumes/j2agent/ui/`。
3. 修改 `.env` 中的 `J2AGENT_VOLUMES_PATH` 后，需将数据迁移到新路径或保持与原挂载一致。
4. MySQL 卷只持久化数据文件；**表结构与种子数据**在 `j2agent` 首次连接空库时由应用自动执行（见 [构建与启动.md](./构建与启动.md#31-数据库初始化首次空库)）。
