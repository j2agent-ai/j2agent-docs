# 图表渲染

## 统一配色

各引擎共用 `src/utils/diagramTheme.ts`：

| 用途 | 说明 |
|------|------|
| `DIAGRAM_COLOR_PALETTE` | 8 色分类色板（`#93C5FD` 起），用于 Mermaid 饼图/xychart、Vega-Lite `range.category` |
| `DIAGRAM_THEME_COLORS` | 文字、轴线、节点底色、连线等语义色 |
| `getMermaidThemeVariables` / `getMermaidThemeCss` | Mermaid 初始化 |
| `getVegaLiteEmbedConfig` | vega-embed 在 quartz 主题上覆盖色板与轴线 |
| `injectPlantUmlTheme` | PlantUML 在 `@start*` 后注入 `skinparam` |

CSS 变量 `--n-color-diagram-palette-1` … `8` 与 TS 色板同步（`var.scss`），`--n-color-diagram-bar` 指向首色。

## Mermaid

### 流程

1. 主线程：`normalizeMermaidSource`（[`diagramSourceNormalize.ts`](../../../j2agent-ui/src/utils/diagramSourceNormalize.ts)）+ `applyXychartDiagramPresentation` 设置 DOM class
2. **Web Worker**：`mermaid.render` 得到 SVG 字符串（[`diagramRender.worker.ts`](../../../j2agent-ui/src/workers/diagramRender.worker.ts)）
3. 主线程：注入 `.md-diagram-body`，xychart 后处理 + `scheduleFitDiagramSvg` 适配尺寸
4. Worker 失败时 fallback：主线程 lazy `import('mermaid')` 走同一规范化逻辑

Worker 路径不执行 `bindFunctions`（静态 SVG）；fallback 路径同样只取 SVG 字符串。

### xychart-beta 与 TOPN 标签重叠

**现象**：竖向柱状图在聊天气泡较窄时，X 轴站点名挤在一起（如「当前告警站点分布 TOP15」）。

**原因**：

1. Mermaid 竖向 `xychart` 对多类目、长文本无旋转/截断（上游 issue #5926 / #7599）
2. `fitDiagramSvgInBody` 曾按宽度等比缩小整张 SVG，文字不重排，叠字更明显

**前端修正**（`markdownRenderer.ts`）：

| 机制 | 条件 | 行为 |
|------|------|------|
| `normalizeXychartOrientation` | `xychart` / `xychart-beta`，类目 ≥ **11**（`XYCHART_HORIZONTAL_MIN_CATEGORIES`），且无 `horizontal` | 首行改为 `xychart-beta horizontal` |
| 宽图模式 | SVG 自然宽度 > 气泡内容区宽度 | 不压窄宽度；`md-diagram-body--wide` + 横向滚动 |

类目标签由 Mermaid 默认横排；**竖向图**（未写 `horizontal`）在渲染后对底部 `g.bottom-axis` 标签做超宽换行（SVG `tspan`），宽度按相邻刻度间距估算；换行后同步扩展 SVG `viewBox` 与气泡容器高度，避免底部被裁切。
| 高度适配 | 始终 | 仍受 `--md-diagram-max-height` 限制 |

Agent 侧推荐在 Skill 中直接写 `xychart-beta horizontal`（类目 ≥ 8），减少依赖自动修正。

### 柱/条颜色（横竖向一致）

- 色板来自 `DIAGRAM_COLOR_PALETTE`，经 `themeVariables.xyChart.plotColorPalette` 与 `pie1`–`pie8` 注入。
- 渲染后对 `g.bar-plot-* rect` 执行 `applyXychartBarColors`，避免竖向图仍走默认紫系。
- CSS 变量 `--n-color-diagram-bar` 作首系列兜底。

### 其它 Mermaid 类型

- **pie**：`quotePieSliceLines` 为未加引号的扇区标签补双引号
- **flowchart / graph**：
  - `normalizeFlowchartSubgraphLabels`：修正 `subgraph入库侧` / `subgraph 查询侧` 为 `subgraph _sgN [标签]`
  - `normalizeFlowchartMultilineNodeDefs`：同行多个节点定义（如 `NE["网元"]  CHASSIS["机框"]`）拆成多行
  - `normalizeFlowchartMultilineEdges`：同行多条串联边（如 `A --> B  B --> C`）拆成多行
- **sequenceDiagram**：依赖主题变量，无额外缩放逻辑

### 错误展示

解析或渲染失败时，块加 `md-diagram-error`，body 内为标题 + `<details>` 错误详情，避免被 `overflow: hidden` 裁切。

## Vega-Lite

- 围栏：`vega-lite` / `vegalite`
- `parseVegaLiteSpec`（`diagramSourceNormalize.ts`）：JSON 规范化 + `enhanceVegaLiteSpec`
- **多类目 X 轴**：`encoding.x` 为 nominal/ordinal，`data.values` 去重类目 ≥ 8，且未配置 `axis.labelAngle` 时，自动 `labelAngle: -45`、`labelLimit: 120`
- **Worker 渲染**：`vega-lite` compile + `vega.View({ renderer: 'svg' }).toSVG()`，主题 `getVegaLiteEmbedConfig()`
- **主线程**：`mountDiagramMarkup` 将 SVG 挂到 `body` 直下 + `scheduleFitDiagramSvg`
- **Fallback**：隐藏 DOM + `vega-embed`（`renderer: 'svg'`）生成 SVG 字符串后同样提到 body 直下

## PlantUML

- **Worker 内**动态加载：`@plantuml/core`（先 `viz-global.js`，再 `renderToString`）
- 渲染前 `injectPlantUmlTheme` 注入统一 `skinparam`
- 主线程注入 body，走同一套 `fitDiagramSvgInBody`
- Worker 不可用时 fallback 至主线程相同逻辑

## HTML 预览

- ` ```html ` → 气泡内为无滚动的等比缩略图（`resizeHtmlPreview` + CSS `scale` 适配 `--md-html-preview-max-height`，默认 `min(680px, 75vh)`）；缩略图 iframe `pointer-events: none`，悬停显示「点击放大」

- 生成中（`md-block-pending`）显示 shimmer + 省略号动画；`md-block-pending` 在 `iframe.onload` 并完成缩略图适配后才移除

## 尺寸与 ResizeObserver

- CSS 变量 `--md-diagram-max-height`（默认 `min(360px, 42vh)`，见 `markdown.scss`）；HTML 预览独立使用 `--md-html-preview-max-height`（默认 `min(680px, 75vh)`）
- `fitDiagramSvgInBody` 读取 viewBox / width / height / getBBox 作为自然尺寸
- `ResizeObserver` 监听气泡、图表块宽度变化；`window.resize` 时失效高度缓存并重算

## 渲染性能

`ChatView.vue` 进入聊天页后通过 `requestIdleCallback`（或 `setTimeout` 兜底）调用 `preloadDiagramRuntimes()`，提前加载 mermaid / vega-embed / PlantUML 运行时，降低首图冷启动延迟。

流式输出时对 `activateMarkdownBlocks` 做约 100ms debounce，减少每个 token 触发的全列表扫描；`isBusyByState` 变为 false 时立即 `flush`，确保流式尾段图表不延迟。

`renderMarkdownBlocks` 约定：

| 机制 | 行为 |
|------|------|
| 待渲染选择器 | 仅查询未渲染或 `data-md-revision` 落后于 `MARKDOWN_RENDERER_REVISION` 的块 |
| 流式 scope | busy 时仅扫描当前轮 assistant 消息的 `.message-bubble-wrap` 子树 |
| 增量 refit | 本次新完成的图表块才 `scheduleFitDiagramSvgReflow`；侧栏拉宽等全局 resize 仍走 `refitDiagramBlocksInRoot` 全量重算 |
| 并发上限 | 默认最多 2 个图表块同时异步渲染，避免多图同时阻塞主线程 |

## 全屏预览

`ChatView.vue`：点击 `.md-diagram-body svg`（非 error 块）收集同气泡内所有 SVG，克隆后由 `DiagramPreviewOverlay` **内联挂载**到 DOM 中切换预览。

**不可**将 SVG 序列化为 blob 后用 `ElImageViewer` 的 `<img>` 展示：Mermaid 默认 `htmlLabels: true`，节点标签通过 `<foreignObject>` 内嵌 HTML 渲染；浏览器把 SVG 当作外部图片加载时不会渲染 `foreignObject` 内容，表现为第二行/中文乱码。气泡内联 DOM 无此限制。

实现要点（`src/utils/diagramPreview.ts` + `DiagramPreviewOverlay.vue`）：

- `cloneSvgForPreview`：去掉 `fitDiagramSvgInBody` 写入的 inline 尺寸，恢复 `viewBox` 原始宽高，铺白底并内联 `DIAGRAM_FONT_FAMILY`
- Markdown 图片与用户附件仍走 `ElImageViewer`
- 图表预览 `DiagramPreviewOverlay` 对齐 `ElImageViewer` 交互：缩放（滚轮/方向键/工具栏）、拖拽平移、旋转、适应/原始尺寸切换、多图左右切换
- 工具栏「保存 SVG」：对当前预览图序列化并下载 `.svg` 文件（`diagramExport.ts`）；多图时文件名带序号

## 为什么改了代码，旧会话里的图「刷新」不变？

**不是 Mermaid 把 SVG 缓存在磁盘**，也**不是**后端把图表存进数据库。消息里只有 Markdown 原文，每次展示都会走 `renderMarkdown` + `renderMarkdownBlocks`。

常见原因有两类：

| 原因 | 说明 | 处理 |
|------|------|------|
| **浏览器 / 部署仍是旧前端包** | 生产环境静态资源在 `${AI_CENTER_BASE_PATH}/ui`，带 hash 的 `assets/*.js` 会被浏览器长期缓存；若未重新 `npm run build` 并同步 `dist`，或只普通刷新，仍执行旧 JS | 重新构建并发布 `dist`；浏览器 **强制刷新**（Cmd/Ctrl+Shift+R）或无痕窗口；见 [前端静态资源更新](../../基础设施/docker部署/前端静态资源更新.md) |
| **SPA 内 DOM 未重建** | `v-html` 输出的 HTML 字符串若与上次相同，Vue 可能不替换已有 DOM；图表块上已有 `data-md-rendered="true"` 时，`renderMarkdownBlocks` 会直接跳过 | 已用 `MARKDOWN_RENDERER_REVISION` + 气泡 `:key`：前端版本递增后，历史消息会强制重绘图表 |

开发环境请确认 `npm run dev` 已热更新；若仍像旧版，重启 dev 并硬刷新页面。
